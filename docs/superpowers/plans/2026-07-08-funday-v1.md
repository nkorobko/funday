# FunDay v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship FunDay v1 — a static, bilingual (he-RTL / en-LTR) fun-day planning app with a curated provider catalog, booking-request CTAs, saved plans, an admin page, and CI — publicly served by GitHub Pages from `main`.

**Architecture:** Pure static site, no build step. ES modules split pure logic (testable in node) from DOM wiring. `data/activities.json` is the database; a node validation script + unit tests run in GitHub Actions. The verified `mockup.html` is the visual/behavioral reference — v1 ports it into modular files and adds catalog loading, validation, error handling, and the admin page.

**Tech Stack:** Vanilla JS (ES modules), CSS, `node:test` for unit tests, GitHub Actions, GitHub Pages.

## Global Constraints

- Static only — must work on GitHub Pages with no server, no secrets (spec: "Zero backend, zero secrets, zero monthly cost").
- No frameworks, no build step, no runtime dependencies. `package.json` exists only for `"type": "module"` and test scripts.
- Bilingual: Hebrew RTL (region `il`, currency ILS/₪) and English LTR (region `dallas`, currency USD/$). Region switch persisted in localStorage.
- Data model per spec rev 2: `id, region, name, category, city, currency, price_per_person, group_min, group_max, description, rating?, reviews_link?, link?, provider{name, phone?, email?}, tags?`.
- Matching rules: category/city exact match when set; group within `[group_min, group_max]`; over-budget results shown, flagged, sorted after in-budget; ties by total (or price/person when no group) ascending.
- Booking CTAs: `https://wa.me/<phone>?text=<prefilled>` and `mailto:` with prefilled subject/body; hide a CTA when its contact field is missing.
- Error handling: catalog fetch failure → message + retry button; invalid entries skipped with `console.warn`; empty results → loosen-filters message.
- Conifers branding per official brand guide (Simplified Brand Guide, Jan 2025): logo `assets/conifers-logo.png`, favicon `assets/conifers-fav32.png`. Tokens: Main Green `#1A2C19` (header/base), Overlay Green `#1F4B1D`, Accent Green `#B6F5B1`, light-mode Accent Green `#243F23`, Accent Orange `#EE7D20` (primary CTA), Overlay Orange `#FFD1B4`, Light BG `#F7EEE9` (body background), Light Grey/Brown `#CDC8BD`, Dark Grey/Brown `#625D50`. Typography: `Aptos` first in the font stack (fallbacks: Segoe UI, Heebo, Arial), Semibold for headings, never Aptos Display.

```js
// Pure logic — no DOM access. Tested by tests/logic.test.mjs and reused
// by app.js, admin.js and scripts/validate-catalog.mjs.

export function annotateResults(activities, { category = "", city = "", budget = 0, group = 0 } = {}) {
  return activities
    .filter(a =>
      (!category || a.category === category) &&
      (!city || a.city === city) &&
      (!group || (group >= a.group_min && group <= a.group_max)))
    .map(a => {
      const total = group ? a.price_per_person * group : null;
      const over = (budget && total) ? Math.max(0, total - budget) : 0;
      return { ...a, total, over, inBudget: !budget || !total || total <= budget };
    })
    .sort((x, y) => (x.inBudget === y.inBudget)
      ? (x.total ?? x.price_per_person) - (y.total ?? y.price_per_person)
      : (x.inBudget ? -1 : 1));
}

export function buildBookingLinks(activity, { group = 0, budget = 0 } = {}, t) {
  const msg = encodeURIComponent(t.waMsg(activity, group, budget));
  const phone = activity.provider?.phone;
  const email = activity.provider?.email;
  return {
    wa: phone ? `https://wa.me/${phone}?text=${msg}` : null,
    mail: email ? `mailto:${email}?subject=${encodeURIComponent(t.mailSubject(activity))}&body=${msg}` : null
  };
}

const REGIONS = { il: "ILS", dallas: "USD" };

export function validateActivity(a) {
  const errs = [];
  if (!a || typeof a !== "object") return ["entry is not an object"];
  const str = f => typeof a[f] === "string" && a[f].trim().length > 0;
  for (const f of ["id", "name", "category", "city", "description"]) {
    if (!str(f)) errs.push(`missing/empty string field: ${f}`);
  }
  if (!(a.region in REGIONS)) errs.push(`unknown region: ${a.region}`);
  else if (a.currency !== REGIONS[a.region]) errs.push(`currency ${a.currency} does not match region ${a.region}`);
  if (!Number.isInteger(a.price_per_person) || a.price_per_person <= 0) errs.push("price_per_person must be a positive integer");
  if (!Number.isInteger(a.group_min) || a.group_min < 1) errs.push("group_min must be an integer >= 1");
  if (!Number.isInteger(a.group_max) || a.group_max < a.group_min) errs.push("group_max must be an integer >= group_min");
  if (a.rating !== undefined && !(typeof a.rating === "number" && a.rating >= 1 && a.rating <= 5)) errs.push("rating must be a number 1..5");
  if (!a.provider || typeof a.provider !== "object" || !(typeof a.provider.name === "string" && a.provider.name.trim())) {
    errs.push("provider.name is required");
  } else {
    if (a.provider.phone !== undefined && !/^\d{8,15}$/.test(a.provider.phone)) errs.push("provider.phone must be 8-15 digits (E.164, no +)");
    if (a.provider.email !== undefined && !/^[^@\s]+@[^@\s]+\.[^@\s]+$/.test(a.provider.email)) errs.push("provider.email is not a valid email");
  }
  return errs;
}

export function validateCatalog(list) {
  if (!Array.isArray(list)) return ["catalog is not an array"];
  const errs = [];
  const seen = new Set();
  list.forEach((a, i) => {
    const id = a?.id ?? `#${i}`;
    if (a?.id) {
      if (seen.has(a.id)) errs.push(`duplicate id: ${a.id}`);
      seen.add(a.id);
    }
    validateActivity(a).forEach(e => errs.push(`[${id}] ${e}`));
  });
  return errs;
}
```

```js
};
```
